Numerical Integration of the time-independent Schroedinger Equation.
A simple example using fortran and PGPLOT graphics.

The time-independent Schroedinger equation of a Harmonic oscillator (a mass m reacting to a spring with spring constant k) is (note: html isn't good at symbols and I am using  h to represent hbar)

-(h2/2m)d2y/dx2 +(kx2/2)y = Ey.
Let us use a new dimensionless variable u = x/x0.  For the moment we don't specify what value the "length scale" x0 should take.  (We know from the classical Harmonic oscillator that there is an intrinsic time scale, the reciprocal of the frequency scale w02 = k/m    .  But the classical problem has no intrinsic length scale.  The properties do not depend on the amplitude of oscillation.)  Then the Schroedinger equation is
-(h2/2mx02)d2y/du2 +(kx02/2)u2y = Ey.
This equation has two factors, (h2/2mx02)  and  (kx02/2) which each have units energy.  The equation simplifies if we specify that  x02should be such that the two factors are equal.
(h2/2mx02)  = (kx02/2)   or   x04= h2/mk or  x04= h2/m2w02or  x0 = (h/mw0)1/2
The harmonic oscillator acquires a length scale when quantum mechanics is added!  Using this choice for x0 ,  the two factors of dimension energy both become
(h2/2mx02)  = (kx02/2)  =  hw0/2.
Thus there is also an intrinsic energy scale, the zero point energy E0=hw0/2.   If we define the dimensionless energy to be e=E/E0, then the Schroedinger equation becomes (after dividing out the common factor E0),
-d2y/du2 +u2y = ey.
The Schroedinger equation can be written as two coupled first-order equations (instead of a single second-order equation):
dy/du = f(u) = (y(u+du)-y(u))/du
df/du = (u2-e)y(u) = (f(u+du)-f(u))/du.
The last part of these equations is exact only in the limit as du goes to zero.  But we will solve these equations for some small value of du, expecting that the answer will be reasonably good if du is small compared to the intrinsic scale of dimensionless length, namely 1.  In the example below, we take du to be .01.  The approximate equations are
y(u+du) = y(u) + f(u) du
f(u+du) = f(u) + (u2-e)y(u) du.
The Fortran program below integrates the equation of motion by sequential application of the 2 equations above.  This is known as the "Euler method" or sometimes as the "forward Euler method" for integrating.  Given initial values, at some initial displacement u0 ( we will choose u0=0, which is the usual choice), the Schroedinger equation determines the values of  the wavefunction y(u) and the derivative f(u)  at all other values of displacement u.  What you discover when you integrate forward or backward in displacement u is that the wavefunction almost always diverges rapidly to + or - infinity.  The wavefunction cannot be normalized because the integral of | y(u) |2 over all u is infinite.  A non-normalizable wavefunction has to be rejected -- it is not a valid stationary state of the time-dependent Schroedinger equation.  However, for special values of energy En (or dimensionless energy en) the wavefunction   decays to 0 at large values of u.  You can locate these special energies (the "eigen-energies" or energy eigenvalues) by trial and error using the program below.  The subscript n just counts the states (or counts the number of nodes or zeros in the wavefunction.)

What initial values should we take for  y(0) and  f(0)? The oscillator has the simplifying feature that the potential energy

     kx2/2=  E0 u2

is an even function of displacement u.  It is therefore possible to choose the wavefunctions  y(u) to be either even or odd functions of u, that is,

      y(-u)= + y(u)   (even)  or  y(-u)   = - y(u)    (odd).

The even solutions have  y(0)=1 and  f(0)=0.  The odd solutions have  y(0)=0 and  f(0) =1.  The choice 1 is arbitrary.  We could choose any non-zero number.  Later, after the full solution  y(u) is known, it can be rescaled to have normalization equal to the conventional value, 1.
 

Finally, a note about the Euler algorithm for integrating the Schroedinger equation.  It is actually not a very good algorithm.  You should verify that the accuracy of the eigenenergies scales as (du) to some particular low power (which power?)  Better algorithms are available.  The most popular is called the "fourth order Runge-Kutta" algorithm.  But for our purposes, the simple Euler algorithm is sufficient.  Computers are so fast that we can afford to choose du much smaller than .01 if we wish.
Sample program

       program Harmoniceigen
c integrate Schrodinger equation to find wavefunction of Harmonic oscillator.
c use simplest possible "forward Euler" integrating algorithm
c use PGPLOT graphics.  Compile statement for SUN workstations at
c the math sinc site:  g77 -o prog prog.f $PGPLOTL
c find energies e at which psi -> 0 as x -> infinity
c use dimensionless variables u=x/a, a=root(hbar/m omega), e=2E/hbar omega
      implicit none
      real phi(2),psi(2),u(2),du,e,phi0,psi0
      integer i,n,PGBEG,ier,PGOPEN
c choose step size delta=du (for example, du=0.01)
      write(6,*) 'what is step size du?'
      read(5,*) du
c choose initial values of psi and phi=d(psi)/dx (eg, 1 and 0
c for even solutions, 0 and 1 for odd solutions)
      write(6,*) 'what is psi(0)?'
      read(5,*) psi0
      write(6,*) 'what is phi(0) (derivative of phi)?'
      read(5,*) phi0
c integrate from u=0 to u=N*du; choose N  (since the graph plots u values
c only out to 6.5, the best choice would be 6.5/du or 650 steps.  The
c setting 6.5 can be changed; it is set in the PGENV call below.)
      write(6,*) 'how many steps?'
      read(5,*) n
      write(6,*) 'what trial energy e?'
      read(5,*) e
c      ier=PGBEG(0,'?',1,1)
c open PGPLOT using xwindow
      ier=PGOPEN('/xwin')
      if (ier .ne. 1) stop
c PGPAP scales the size and aspect ratio of the plot
      call PGPAP(7.0,1.0)
c PGENV chooses beginning and end of x and y axis
      call PGENV(-0.5,6.5,-2.5,2.5,1,1)
c PGLAB labels the axes and the graph
      call PGLAB('dimensionless position','wavefunction amplitude',
     1 'Harmonic Oscillator')
        do while (e .gt. 0.)
          psi(1)=psi0
          phi(1)=phi0
          u(1)=0.
          do 100 i=1,n
            u(2)=u(1)+du
            psi(2)=psi(1)+phi(1)*du
            phi(2)=phi(1)+(u(1)*u(1)-e)*psi(1)*du
c PGLINE plots the next increment of psi versus u.
            call PGLINE(2,u,psi)
            u(1)=u(2)
            psi(1)=psi(2)
            phi(1)=phi(2)
  100     continue
          write(6,*) 'another e?'
          read(5,*) e
c to stop, use e<0.
        end do
      call PGEND
      end
