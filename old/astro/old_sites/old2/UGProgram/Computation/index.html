Most people agree that there is a place in the undergraduate curriculum for some computation, but there are many shades of opinion about what that means.  Here are two notions which are relatively non-controversial.

(1) Physics and Astronomy students are trained to solve problems.  If the computer is the appropriate tool, students should be able to use the computer to solve the problem.  Usually this will mean writing a code, which first involves breaking the problem down to a sequence of computational steps.

(2) The core physics topics like mechanics, electrodynamics, statistical mechanics, and quantum mechanics, are the foundations of physics instruction.  Provided numerical computation enhances one's understanding, then there is a benefit in including some computation in coursework.

With these notions in mind, the instructors of some of the core undergraduate courses have agreed to add some computation as a pedagogical experiment.  Students with a prior knowledge of a computing language like Fortran, C, or C++, can use the language they prefer, but we will choose Fortran as the default language, primarily because it is easy to learn and use.  We will also incorporate graphics directly into the codes using the PGPLOT graphics package.  This package will be available on the SUN machines in the Math sinc site.  Getting started requires learning a few things:

1. operating system.  If you wish to use your own computer resources, fine.  The default option is the SUN computers which run a version of unix.  You don't need much expertise with unix, but need to know a few basics.

2. How to break down the problem into computational steps.  There are two aspects, (a) choosing appropriate units for the computation, and (b) using an appropriate method of numerical analysis.  You do not need to be expert with numerical analysis, but you do need to know how to test your results to evaluate the accuracy of your answers.

3.  A language to write the code in, and graphics to see the result.  Fortran and PGPLOT are a nice way to go, but use your own language if you wish.  Symbol manipulating languages like MAPLE, Mathcad, and Mathematica, are valuable tools but not appropriate for the instructional purposes at hand.  One of the benefits of learning to write your own code is that it provides a basis for evaluating the results of packaged programs.
